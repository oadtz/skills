# Trend & problem sources — Step 2

Generate ideas from real signal. Run several searches **in parallel**, mixing two kinds of source:
*authority/trend* sources (where the puck is going) and *problem/complaint* sources (pain in
people's own words). Then ground both in the user's specific domain.

## Authority / trend sources
- **Y Combinator — Requests for Startups** (`ycombinator.com/rfs`): what top investors actively
  want built right now. Re-fetch each run — it changes.
- **a16z Big Ideas** (annual, per-team): forward bets across apps, infra, bio, American Dynamism,
  crypto.
- **Investor / operator essays**: Paul Graham essays, first-principles pieces, vertical-specific
  analysts.
- **Earnings calls & industry reports** for the user's domain (what big players say is breaking).

## Problem / complaint sources (pain in real words)
- **Reddit** — niche subreddits for the domain; search `site:reddit.com [domain] "i wish"`,
  `"is there a tool that"`, `"how do you all deal with"`.
- **Hacker News** — "Ask HN" threads; pain points and "Show HN" gaps.
- **GitHub issues** — unmet feature requests; quantify (`N issues asking for X`).
- **Review sites** (G2, Capterra, app stores, Trustpilot) — 1–3 star reviews reveal what existing
  tools fail at.
- **Q&A / forums** — Quora, Stack Exchange, domain-specific forums.
- **Job postings** — roles a company is hiring for hint at processes ripe for automation-as-service.

## Connected tools (use if available)
If the user has connected enterprise/data tools, mine them for first-party pain: support tickets,
sales-call notes, internal docs, analytics. First-party signal beats public signal — it's pain the
user can already see and verify.

## Ready-made query patterns
Swap `[domain]` for the user's industry/segment:
- `[domain] biggest pain points 2026`
- `site:reddit.com [domain] "i hate" OR "i wish there was"`
- `[domain] software "switched from" OR "looking for alternative"`
- `[domain] manual process spreadsheet time-consuming`
- `"Ask HN" [domain] problem`
- `[domain] startup ideas Y Combinator request`
- `[incumbent tool] complaints OR limitations OR "doesn't support"`

## How much to gather
Enough to ground 10–20 raw ideas across your diversity partitions — typically 5–8 searches. Capture
the painful quotes and quantified signals verbatim before interpreting; they become the evidence
trail behind each idea and the raw material for the "Secret" and "Job" scoring dimensions.
